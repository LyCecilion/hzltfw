# 发布说明

[English](RELEASE_NOTES.md)

## v1.0.0

`v1.0.0` 是 Hazelita Forensics Workbench 的第一个大作业正式交付版本。
该版本面向可重复的本地课堂演示，不试图在第一版做成完整商业取证套件。

### 发布范围

- 提供本地 NiceGUI 工作流：创建案件、导入检材、扫描文件、运行插件、查看
  artifact、导出报告。
- 内置 hash 清单、文件类型伪装检测、关键词/正则命中、压缩包索引和部分
  元数据提取插件。
- 提供可选 ALEAPP、iLEAPP、Hindsight 外部工具适配。外部工具命令通过
  `.hzltfw/config.json` 配置；外部工具源码树和二进制文件不进入本仓库。
- 提供面向发现的 artifact 查看界面，对常见结果使用专门视图，原始 JSON
  收进高级视图。
- 主界面支持中文/英文；插件细节允许回退到英文。
- 支持 Markdown 报告导出和可携带报告包导出。报告包会复制已链接的外部工具
  输出，便于提交或在另一台机器上打开。

### 推荐演示流程

1. 创建案件。
2. 添加准备好的目录或压缩包检材。
3. 扫描检材。
4. 查看文件索引。
5. 运行内置插件。
6. 如检材和本机工具配置满足条件，可运行 ALEAPP、iLEAPP 或 Hindsight。
7. 在发现中心查看结果。
8. 导出报告包用于提交或展示。

### 打包与交付说明

- 开发环境使用 `uv sync --dev`。
- 本地运行使用 `uv run hzltfw`。
- 外部取证工具保留在仓库之外，并通过 `.hzltfw/config.json` 配置命令。
- 不要提交样本检材、外部工具输出或 `.hzltfw/` 运行时工作区数据。

### 已知边界

- 检材需要先由用户导出为文件、目录或压缩包。原始磁盘镜像解析、E01 解析、
  分区解析和文件系统重建不属于本版本范围。
- 外部工具适配器保留并链接外部报告，不会把外部工具的所有发现完整导入统一
  artifact 数据库。
- 压缩包能力保持轻量：当前关注 ZIP 条目索引，不把压缩包解压和递归分析作为
  发布要求。
- Chromium History 解析仍是加分项，不应阻塞主演示流程。
